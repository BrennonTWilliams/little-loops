# FEAT-2088: distill-traces: core skill implementation and tests

## Summary

Implement the `distill-traces` skill that mines successful loop execution traces from `.loops/.history/` into reusable YAML state templates and transition fragments, and write section-scoped tests to verify its structure.

## Parent Issue

Decomposed from FEAT-2078: Add distill-traces skill to extract reusable loop fragments from history

## Current Behavior

No `distill-traces` skill exists. Reusable loop patterns must be identified manually by inspecting `.loops/.history/` run artifacts and hand-authoring YAML fragments from scratch.

## Expected Behavior

Running `/ll:distill-traces <loop-name>` reads qualifying successful run artifacts from `.loops/.history/`, aggregates common state sequences and action strings, and writes YAML fragment templates to `scripts/little_loops/loops/lib/<loop-name>/` alongside a human-readable `primitives.md` index.

## Use Case

A loop author wants to seed a new loop with proven patterns from an existing, well-exercised loop (e.g. `rn-remediate`). They run `/ll:distill-traces rn-remediate --min-success 5`, get a `state-templates.yaml` and `transitions.yaml` in `scripts/little_loops/loops/lib/rn-remediate/`, then reference those fragments in their new loop via `import: - lib/rn-remediate/state-templates.yaml` — skipping the trial-and-error phase of building evaluator and action patterns.

## Proposed Solution

### Step 1 — Create `skills/distill-traces/SKILL.md`

Model after `skills/debug-loop-run/SKILL.md`. Include:
- `arguments:` frontmatter block with `disable-model-invocation: true`, `argument-hint:`, and `metadata.short-description:` fields
- Skill name: `distill-traces`
- Arguments: `<loop-name> [--min-success N]`

### Step 2 — Run discovery

Glob `.loops/.history/` using the `_RUN_FOLDER` regex from `scripts/little_loops/fsm/persistence.py`:
```
r"^(\d{4}-\d{2}-\d{2}T\d{6})-(.+)$"
```
Filter candidates where the run name ends with `-<loop_name>`. For each candidate, read `state.json` and keep only runs where `status == "completed"` and `terminated_by == "terminal"`. Apply `--min-success N` threshold (default 3).

### Step 3 — Extract state sequences from `events.jsonl`

For each qualifying run, collect:
- `state_enter` events (field: `state`)
- `route` events (fields: `from`, `to`)

Aggregate most-frequent sequences and action strings across runs.

### Step 4 — Serialize fragments YAML

Output YAML must follow the `fragments:` map structure from `scripts/little_loops/loops/lib/common.yaml`:

```yaml
# Generated by /ll:distill-traces <loop-name>
# Import with: import: - lib/<loop-name>/state-templates.yaml
fragments:
  <state_name>:
    description: |
      Extracted from N successful runs of <loop-name>.
      Caller must supply: on_yes, on_no (and optionally on_error, timeout).
    action_type: shell|prompt|slash_command
    action: |
      <most common action string across qualifying runs>
    evaluate:
      type: exit_code|llm_structured|output_contains
```

Write to:
- `scripts/little_loops/loops/lib/<loop-name>/state-templates.yaml`
- `scripts/little_loops/loops/lib/<loop-name>/transitions.yaml`

Use `mkdir -p` + Write (pattern from `commands/loop-suggester.md` Step 7).

### Step 5 — Write `primitives.md`

Write `scripts/little_loops/loops/lib/<loop-name>/primitives.md` as a human-readable index of extracted fragments with run counts and frequency data.

### Step 6 — Graceful degradation

If fewer than `--min-success` successful runs exist, print a clear message and exit 0 without writing artifacts.

### Step 7 — Tests

Write `scripts/tests/test_distill_traces_skill.py` following the section-scoped assertion pattern from `scripts/tests/test_audit_loop_run_skill.py`. Assert:
- Skill file exists
- `--min-success` is documented
- `state.json`/`events.jsonl` references appear
- Fragment schema section is present

Use fixture shapes from:
- `scripts/tests/test_ll_loop_commands.py:TestCmdNextLoop.loops_dir_with_history` — mock `.history/` run dirs with `state.json` and `events.jsonl`
- `scripts/tests/test_loop_run_analytics.py:_make_events_jsonl` — canonical `state_enter`/`route` event shapes
- `scripts/tests/test_fsm_persistence.py:TestArchiveRun` — `LoopState` fixture shape and timestamp format (`<YYYY-MM-DDTHHMMSS>-<loop_name>`, e.g. `"2024-01-15T103000-test-loop"`)
