# FEAT-2393: Documentation migration and alias retirement for ll-adapt

## Summary

Update all documentation, contributor guides, and generated files to reference the unified `ll-adapt --host <host>` interface instead of the legacy per-host scripts. Regenerate all 9 `.codex/agents/*.toml` files. **Depends on FEAT-2391** (requires `ll-adapt --host codex --apply` to be runnable before TOML regeneration).

## Parent Issue

Decomposed from FEAT-2260: Generic host-parameterized skill + command adapter

## Current Behavior

All documentation, contributor guides, Codex getting-started docs, and CLI reference pages reference the legacy `ll-adapt-skills-for-codex` and `ll-adapt-agents-for-codex` scripts. The `.claude/CLAUDE.md` CLI Tools section lists two separate bullets for these legacy commands. The `.codex/agents/*.toml` files carry the old generation marker and were produced by the superseded per-host scripts.

## Expected Behavior

All documentation references the unified `ll-adapt --host <host>` interface. The `.claude/CLAUDE.md` CLI Tools section shows a single `ll-adapt --host <host>` bullet. All 9 `.codex/agents/*.toml` files have first-line marker `# generated by ll-adapt`. `TestRealAgentsIntegrationGuard` and the full test suite pass after regeneration.

## Use Case

A contributor adding Codex support for a new skill reads `docs/codex/getting-started.md` and runs `ll-adapt --host codex --apply` to regenerate adapter files. The unified interface is the only documented path — no legacy scripts to disambiguate between.

## Motivation

The unified `ll-adapt --host <host>` adapter introduced in FEAT-2391 supersedes the per-host scripts (`ll-adapt-skills-for-codex`, `ll-adapt-agents-for-codex`). Leaving legacy references in docs creates confusion for contributors, risks new contributors using deprecated entry points, and leaves `.codex/agents/*.toml` files out-of-sync with the new generation marker. Retiring alias documentation ensures the unified interface is the single authoritative path.
