# FEAT-2190: GEMINI.md project context file — ll:init --gemini

## Summary

Extend `ll-init` to support `--gemini` host selection, which generates a
`GEMINI.md` file in the project root. `GEMINI.md` is the exact analog of
`CLAUDE.md` for Gemini CLI — Gemini loads it as project instructions on startup.

From FEAT-2179: Gemini uses `GEMINI.md` as its project instructions file
(confirmed; exact analog of `CLAUDE.md`).

## Use Case

A user initializes little-loops in a Gemini project: `ll-init --hosts gemini`.
`ll-init` generates `GEMINI.md` with the same little-loops project instructions
that `CLAUDE.md` provides for Claude Code users.

## Implementation Steps

1. Add `gemini` to `--hosts` multi-select in `ll-init` CLI
   (`scripts/little_loops/init/`).
2. Add a `GEMINI.md` template in `templates/` (or generate from `CLAUDE.md`
   template by swapping Claude-specific references).
3. Wire `--hosts gemini` to generate `GEMINI.md` from the template.
4. If the project already has a `GEMINI.md`, emit a warning and skip
   (don't overwrite user customizations).
5. Update `ll-init --dry-run` output to show `GEMINI.md` as a planned file.
6. Add tests in `scripts/tests/test_init.py`.

## Acceptance Criteria

- `ll-init --hosts gemini` generates `GEMINI.md` with ll project instructions.
- Re-running with an existing `GEMINI.md` does not overwrite (warns instead).
- `ll-init --dry-run --hosts gemini` shows `GEMINI.md` in the plan.
- Tests pass.

## API/Interface

### New Files

- `templates/GEMINI.md.j2` (or equivalent template)

### Files to Modify

- `scripts/little_loops/init/` — `--hosts gemini` branch
- `scripts/tests/test_init.py` — Gemini init coverage

## Research Notes (FEAT-2179)

loaded automatically on session start.
