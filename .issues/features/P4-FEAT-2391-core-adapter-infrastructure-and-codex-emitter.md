# FEAT-2391: Core adapter infrastructure + CodexEmitter

## Summary

Build the shared adapter core (`adapters/core.py`, `HostEmitter` protocol, registry factory) and migrate the existing Codex adaptation logic into `CodexEmitter`. Wire the unified `ll-adapt --host <host>` CLI entry point and update internal integration points (permissions, host_runner warning, wiring tests).

## Parent Issue

Decomposed from FEAT-2260: Generic host-parameterized skill + command adapter

## Motivation

Two separate scripts (`ll-adapt-skills-for-codex`, `ll-adapt-agents-for-codex`) duplicate traversal logic and carry a latent behavioral gap: the `disable-model-invocation` filter is applied to skills but silently skipped for commands. As additional host targets are added under EPIC-2257, each would require yet another pair of standalone scripts. Centralizing into a `HostEmitter` protocol + registry factory (mirroring the `HostRunner` pattern) eliminates the duplication, closes the filter gap, and gives future hosts a clear single-extension-point contract.

## Use Case

A developer enabling Codex CLI support for their project runs `ll-adapt --host codex` once to generate all Codex-compatible skill, command, and agent definitions. Previously they ran `ll-adapt-skills-for-codex` and `ll-adapt-agents-for-codex` separately. The new command's `--apply` flag writes output in place; `--dry-run` previews changes before committing.

## Current Behavior

Two standalone scripts handle Codex adaptation independently with duplicated traversal logic. The `disable-model-invocation` filter applies to skills but not commands. There is no unified `ll-adapt --host <host>` entry point; adding a new host requires authoring separate scripts rather than implementing a single `HostEmitter` class.

## Expected Behavior

A unified `ll-adapt --host <host>` CLI dispatches to host-specific emitters via a shared `HostEmitter` protocol registered in a factory. The `disable-model-invocation` filter applies consistently to both skills and commands. Adding a new host requires only implementing one `HostEmitter` subclass and registering it — no new CLI scripts needed.

## Acceptance Criteria

- `scripts/little_loops/adapters/core.py` implements shared traversal, `disable-model-invocation` filter (normalized to skip both skills and commands, fixing the existing behavioral gap), `HostEmitter` protocol (`@runtime_checkable`), and `resolve_emitter(host: str) -> HostEmitter` factory.
- `scripts/little_loops/adapters/codex.py` implements `CodexEmitter` with all three `emit_skill`, `emit_command`, `emit_agent` methods, including ENH-2121 rich TOML fields (`sandbox_mode`, `model_reasoning_effort`, `mcp_servers`, `skills.config`) derived from `tools:` frontmatter when available.
- `ll-adapt --host codex` output for skills and agents is byte-for-byte compatible with the existing `ll-adapt-skills-for-codex` / `ll-adapt-agents-for-codex` output (modulo the marker string changing to `"# generated by ll-adapt"`).
- `scripts/little_loops/cli/adapt.py:main_adapt()` is wired into `pyproject.toml` as `ll-adapt`; `cli/__init__.py` re-exports it.
- Old scripts (`ll-adapt-skills-for-codex`, `ll-adapt-agents-for-codex`) remain as thin `--host codex` aliases (not removed in this PR).
- `scripts/little_loops/init/writers.py` `_LL_PERMISSIONS` updated to `Bash(ll-adapt:*)`.
- `scripts/little_loops/host_runner.py:443` `CapabilityNotSupported` warning updated to reference `ll-adapt --host codex`.
- Tests in `scripts/tests/test_adapters.py` cover: shared core traversal, `disable-model-invocation` filter (skills + commands), `CodexEmitter` per-field emit, rich Codex agent TOML fields present when derivable and omitted otherwise, marker string `"# generated by ll-adapt"`, idempotency, user-authored file protection, `--only` filter.
- `scripts/tests/test_wiring_cli_registry.py` updated with `ll-adapt` 3-tuples for `commands/help.md`, `docs/reference/CLI.md`, `.claude/CLAUDE.md`.
- `python -m mypy scripts/little_loops/` passes clean.
