## Problem

`docs/guides/AUTOMATIC_HARNESSING_GUIDE.md` has three accuracy/completeness gaps found during audit.

## Findings

### 1. `check_mcp` not surfaced by the wizard (Accuracy, Medium) ‚Äî lines 13, 284‚Äì299

The guide states "Each harness applies up to five evaluation phases" (line 13) and Step H3 (line 284) says the wizard "presents only relevant options. All available phases are pre-selected." This implies the wizard offers all 5 phases including `check_mcp`.

In reality, `skills/create-loop/loop-types.md` Step H3 only presents **4 options** (Tool-based gates, LLM-as-judge, Diff invariants, Skill-based evaluation) ‚Äî no `check_mcp`. The `check_mcp` phase must be added manually after wizard generation. The guide never says this.

**Fix**: Add a note in Step H3 clarifying that `check_mcp` is not wizard-generated and must be inserted manually into the output YAML, with a pointer to the `check_mcp` section.

### 2. `check_stall` example missing `skip_item` state definition (Completeness, Low) ‚Äî lines 458‚Äì468

The `check_stall` YAML example ends with `on_no: skip_item` but never shows the `skip_item` state that must exist in the FSM. Without it the FSM is invalid. `reference.md` (line 718) shows:

```yaml
skip_item:
  action: echo "Skipping ${captured.current_item.output} after retries"
  action_type: shell
  next: discover
```

**Fix**: Append a `skip_item` state definition to the `check_stall` YAML example, or add a note pointing users to define it.

### 3. `check_stall` placement note inconsistent with YAML example (Accuracy, Low) ‚Äî lines 456‚Äì467

Placement note says: *"Insert `check_stall` between `execute` and the first check state (or between `check_invariants` and `advance`)."*

The YAML example shows `on_yes: advance` ‚Äî only valid for the second placement. The first placement would need `on_yes: check_concrete` (or the first check state).

**Fix**: Show both placement variants or align the placement note with the single example shown.

## Acceptance Criteria

- [ ] Step H3 section notes that `check_mcp` must be added manually (not generated by wizard)
- [ ] `check_stall` YAML example includes `skip_item` state definition or prose note
- [ ] `check_stall` placement note is consistent with the YAML example routing
